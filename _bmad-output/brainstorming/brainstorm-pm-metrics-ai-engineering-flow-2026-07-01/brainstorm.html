How Might We

Divergent Framing
Question · Coach

HMW get story-level PM metrics captured as a byproduct of AI-driven dev flow without double-entry?

Question · User

HMW keep estimates/velocity meaningful when AI accelerates delivery and makes sprint pace unpredictable?

Idea A · Coach

A story manifest file (.story.yaml) created at kickoff — carries PM metadata, read by downstream events.

Idea B · Coach

Git hooks silently extract PM signals from branch names, commits, and PR timestamps — zero prompts.

Idea C · Coach

The AI/SDD agent narrates its own status (design / coding / testing / done) as structured events — acting as a PM proxy.

Decision · User

Chose the combination of Idea A + B + C as the direction to develop.

Decision (correction) · User

Story points should be auto-calculated via defined rules (not manually entered) since code is now LLM-generated. Rules TBD. Direction confirmed: A+B+C.

Insight · User

Leadership's real job: capture PM metrics + Engineering metrics + cost of story points + token cost, per story, rolled up per developer and per project overall.

Decision · User

Completion trigger: the openspec/speckit close/archive command (e.g. opsx archive) is the moment all 4 metric families get calculated and snapshotted.

2

Role Playing

Perspective Shift
Idea · User

Tiered flow: skip the heavy metrics-capture flow for small/low-complexity tasks; apply it for complex tasks where the overhead is worth it.

Insight · User

Developer effort/time-on-task is hard to isolate because they multi-task and multi-prompt across stories concurrently — yet we still need to estimate time spent per story and its completion point.

Decision · User

Complexity classification: the LLM estimates complexity at story initiation using project-defined rules, plus a ~30% manual-activity buffer for code review, unit testing, local feature testing.

Decision (simplifying assumption) · User

For now: developer focuses on one single story at a time — multi-tasking time-tracking is deferred, unanswered.

Idea · Coach

Leadership dashboard should surface a trend metric: token-cost-per-story-point over time, to show whether AI investment is improving delivery efficiency.

3

Constraint Roulette

Forced Constraints

Constraint: capture metrics without modifying openspec/speckit internals. Three survival strategies rolled out:

Option 1 — Wrap the CLI. Intercept openspec/speckit CLI invocations without touching internals.
Option 2 — Custom skills. Build skills that take developer-facing inputs directly as part of this flow.
Option 3 — Claude Code hooks. Auto-trigger on tool-use/stop events to capture metrics without touching openspec/speckit source.

Unsolved · Deferred Question

When a developer multi-tasks and multi-prompts across several stories concurrently, how do we attribute time-on-task to a single story? The session's working assumption — one story at a time — sidesteps this rather than solving it. Session-slice time attribution remains an open problem for a future session.

Synthesis — The Pipeline

Final Convergence

A single pipeline emerged from all five techniques: a complexity gate decides whether the heavy flow even engages, human bookends mark start and finish, silent capture runs invisibly in between, and the close command triggers a full metrics snapshot that rolls up per developer and per project.
