- (decision) Paradigm: event-sourced convergence. Git hooks, Claude Code hooks, and the CLI wrapper each emit events to an intermediary; nothing writes .story.yaml directly. Prevents: 3 independent writers racing/corrupting one shared mutable file.
- (decision) Intermediary: append-only local event log file (e.g. .story-events.jsonl) in-repo, not a daemon or direct remote write. Rule: hooks/agent only ever append events locally; no network dependency at capture time.
- (decision) Contract boundary: snapshot schema (versioned) is the ONLY contract between local capture and central presentation layer; raw event log never crosses that boundary.
- (decision) Source-of-truth adapter pattern: one normalized adapter interface (returns points/goal/sprint/description regardless of backend). Project-level config (alongside project-context.md) declares source_of_truth: jira|confluence|docs-only ONCE per project; kickoff skill reads config, never re-asks per story which source-of-truth applies.
- (decision) Story identity: the .story.yaml manifest (created by the kickoff skill) is the sole source of the story ID; hooks/CLI-wrapper read it from the manifest rather than inferring from branch name/PM-tool convention, since not every project has JIRA/git-branch conventions. Manifest's story ID is confirmed with the developer at kickoff.
- (question by user) Story-point auto-calc rules: user has an old document to reference (attachment not yet received) — most analysis/design/coding/unit-testing/reviews now performed by AI based on requirements complexity and type of work
- (decision) Story-point auto-calc rules (resolves prior open question): Phase-1 estimate computed from openspec artifact state alone before code exists - task count from tasks.md (base points, 30+ tasks triggers must-split), volatility score from openspec stage maturity (explore-only=+5 down to proposal+specs+design=0), novelty modifier from pattern-match against prior .story.yaml records (first-time x1.5 / standard x1.0 / existing-pattern-reuse x0.8). Phase-2 reconciliation at opsx archive computed from the event log - human review cycles (UserPromptSubmit follow-up count or PR review rounds), actual decision points (agent self-narrated 'decision' events), verification complexity (test files x testing-type weight: unit x0.5/integration x1/manual-QA x1.5/perf x2), context management (files/components touched). Variance (estimate vs actual) is logged, never silently overwritten.
- (decision) Multi-tasking time attribution: explicit active-story pointer (.active-story local state file), auto-updated on git checkout (via hook) and Claude Code SessionStart, not a manual command. Switching the pointer closes the outgoing story's time slice and opens the new one; idle timeout (~15min of no PostToolUse/prompt activity) auto-pauses the active slice. Assumes branch-per-story convention; manual override needed if a dev works multiple stories on one branch (open risk, not yet confirmed against real team workflow).
- (decision) Confirmed: branch-per-story convention is a team standard (one branch = one story = one active context). The .active-story pointer design from the multi-tasking decision holds without a manual-override escape hatch for now.

- (decision) AI-tool capture adapter pattern (new AD-10, mirrors AD-4): resolves the multi-AI-tool coverage gap. Event namespace generalizes from claude.* to ai.<tool>.* (amends AD-1a). One normalized 'AI activity' shape (session_start, session_end, activity_count, token_cost) is emitted by a tool-specific adapter per AI tool (Claude Code, Cursor, Copilot, Gemini, etc.); fields the tool cannot report (e.g. Copilot exposes no per-token cost) are explicitly null-with-reason, never defaulted to zero. - (decision) Implementation language/runtime: Python 3.8+ via uv (single-file scripts, uv run, no venv management) for all hook scripts, the opsx CLI wrapper, and the snapshot assembler. Ratifies the existing convention already used by _bmad/scripts/*.py in this repo. Git-invoked hooks in .git/hooks/ are thin shell/batch shims that call 'uv run tools/hooks/git/<name>.py' since git hooks must be directly executable. Rejected: Node.js (new runtime dependency, no existing convention here) and Bash-only (awkward for snapshot-assembler JSON reduction, weaker Windows support).
