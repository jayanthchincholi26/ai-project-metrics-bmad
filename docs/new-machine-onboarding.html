Onboarding a New Machine

What Happens When a Developer Sets This Up

Two different rhythms: a one-time install per machine, and a repeating loop per story afterward. The install is the only manual step a developer ever performs on purpose.

Once per machine

Getting Set Up

A few minutes, the first time this developer touches the project.

1

Clone the repository

Same as joining any project — no new tool to install separately, no account to create.

2

Run tools/setup-hooks once

A single committed script installs the capture hooks into this developer's local git and Claude Code configuration. It's the same script every developer runs — nobody hand-configures their own machine.

3

Done — capture is now live on this machine

From this point on, every commit and every AI coding session this developer makes is captured automatically. Nothing else to remember.

then, for every story
Repeats every story

Working Normally

Confirm at the start, work as usual, close at the end.

1

Start the story

A short check-in confirms points, goal, and sprint — auto-filled already if the project uses JIRA or Confluence.

2

Code normally, with AI

Commits happen. AI coding sessions happen. Nothing extra to type or track.

captured silently in the background — invisible to the developer
3

Close the story

One command, plus a one-line "what actually happened" note.

4

A metrics snapshot appears — automatically

Points estimated vs. actual, time on task, AI cost — assembled the moment the story closes, with nothing the developer had to compile by hand.

↻ Every new story repeats this loop — setup from Phase 1 never has to happen again on this machine.
Bottom lineOne short setup command, once, per developer machine. Everything after that is the developer's normal workflow — the metrics are a side effect, not a separate task.
