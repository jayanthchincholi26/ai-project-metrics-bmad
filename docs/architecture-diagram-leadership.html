System Architecture

Project Metrics That Track Themselves

Everything a developer already does — commits, AI coding sessions, closing out a story — already contains the signal leadership needs. This system catches that signal as it happens instead of asking anyone to re-type it into a tracker afterward.

01 · always on

Git Activity

Commits, branch switches, merges — captured the instant they happen.

02 · always on

AI Coding Sessions

Claude Code session start/end, tool use, prompt activity — captured silently.

03 · at story close

Story Lifecycle

The one command a developer runs to close a story out.

local activity log

One File, One Machine

Every signal above lands as one line in a local log on the developer's own laptop. Nothing is sent anywhere yet — and nothing here can corrupt or overwrite anything else.

→
at story close

Metrics Snapshot

The whole log is turned into one clean, permanent record: points estimated vs. actual, time on task, AI cost. Closing a story again never erases history — it adds a new dated version.

→
not yet built

Leadership Dashboard

Rolls snapshots up across developers and projects. Only ever receives the finished snapshot above — never the raw activity log.

What the developer notices: nothing extra. No timers to start, no status meeting to prep for, no spreadsheet to update.
Why it's trustworthy

Nothing gets silently overwritten

Every time a story closes, a brand-new dated snapshot is created. A correction or a re-close never erases what leadership already saw — the full history is always there to audit.

Why it's low-risk

Runs entirely on the laptop

No new server, no new service to keep running, nothing that depends on the network. The only thing that ever leaves the developer's machine is the finished snapshot — never the raw activity detail.

Why it scales

One pattern, any tool

Works today with JIRA, Confluence, or no PM tool at all, and with Claude Code. The same shape is designed to extend to other AI coding tools without redesigning anything upstream.

Bottom lineMetrics fall out of work developers are already doing. Nobody types a number in twice, and no number in the history can quietly change after the fact.
